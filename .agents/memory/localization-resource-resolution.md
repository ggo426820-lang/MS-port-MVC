---
name: ASP.NET Core localization resource resolution
description: How ResourcesPath and marker class namespace interact to resolve .resx embedded resources
---

The rule: marker classes must be in the **root namespace** (e.g. `MostafaSaidPortfolio`), not a sub-namespace like `MostafaSaidPortfolio.Resources`.

**Why:** With `ResourcesPath = "Resources"`, ASP.NET Core's `ResourceManagerStringLocalizerFactory` builds the base name as `{RootNamespace}.{ResourcesPath}.{TypeSimpleName}`. If the marker class is also in `MostafaSaidPortfolio.Resources`, it looks for `MostafaSaidPortfolio.Resources.Resources.SharedResource` — double "Resources" — which matches nothing. The embedded resource name generated by the SDK is `{Namespace}.{ClassName}` derived from the `.cs` file's namespace, not the `.resx` file path.

**How to apply:** When `ResourcesPath = ""` (empty), the localizer searches for `{Namespace}.{ClassName}` directly. With marker classes in `MostafaSaidPortfolio` namespace, the embedded name `MostafaSaidPortfolio.SharedResource` matches exactly. Keep `.resx` files in `Resources/` folder for organization — the SDK still picks them up as EmbeddedResource automatically (no explicit `<EmbeddedResource>` ItemGroup needed; adding one causes NETSDK1022 duplicate error).

Confirmed working in this project: `ResourcesPath = ""` + marker classes in `namespace MostafaSaidPortfolio` + `.resx` files in `Resources/` folder.
